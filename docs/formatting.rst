Text Formatting
===============

PyMax поддерживает базовое Markdown- и HTML-подобное форматирование при
отправке сообщений. Форматирование применяется автоматически в
``send_message()``, ``Message.answer()``, ``Message.reply()`` и
``Chat.answer()``: библиотека удаляет служебные маркеры из текста и передает
Max список ``Element``.

.. code-block:: python

   from pymax import ParseMode
   from pymax.formatting import Bold, FormattedText, Link, Text, TextFormat

   await message.answer(
       "**Важное:** проверьте [документацию](https://example.com)"
   )

   await message.answer(
       '<b>Важное:</b> проверьте <a href="https://example.com">документацию</a>',
       parse_mode=ParseMode.HTML,
   )

   await message.answer(
       Text(
           "Проверьте ",
           Bold("важное"),
           " и ",
           Link("документацию", url="https://example.com"),
       )
   )

   await message.answer(
       FormattedText(
           "Важное: проверьте документацию",
           [
               TextFormat.bold(0, 7),
               TextFormat.link(18, 12, "https://example.com"),
           ],
       )
   )

Получатель увидит текст без маркеров, а фрагменты будут оформлены средствами
клиента Max.

Если текст нужно отправить без разбора, передайте
``parse_mode=ParseMode.DISABLED``.

По умолчанию используется ``parse_mode=ParseMode.MARKDOWN``. Другие режимы
имеют смысл только для обычных строк: ``Text(...)`` и ``FormattedText(...)``
уже содержат готовое форматирование, поэтому их нельзя смешивать с
``ParseMode.HTML`` или ``ParseMode.DISABLED``.

Для нового кода удобнее использовать строитель форматированного текста:
``Text(...)`` можно смешивать с ``Bold(...)``, ``Link(...)`` и другими
элементами без ручного подсчета ``from`` и ``length``.

Поддерживаемый Markdown-синтаксис
---------------------------------

.. list-table::
   :header-rows: 1
   :widths: 30 30 40

   * - Синтаксис
     - Тип элемента
     - Описание
   * - ``**текст**`` или ``__текст__``
     - ``STRONG``
     - Жирный текст.
   * - ``_текст_`` или ``*текст*``
     - ``EMPHASIZED``
     - Выделенный текст.
   * - ``++текст++``
     - ``UNDERLINE``
     - Подчеркнутый текст.
   * - ``~~текст~~``
     - ``STRIKETHROUGH``
     - Зачеркнутый текст.
   * - ```код```
     - ``MONOSPACED``
     - Моноширинный фрагмент в строке.
   * - ``[текст](https://example.com)``
     - ``LINK``
     - Ссылка с отображаемым текстом.
   * - ``# Заголовок``
     - ``HEADING``
     - Заголовок в начале строки.
   * - ``> цитата``
     - ``QUOTE``
     - Цитата в начале строки.

Блоки кода
----------

Многострочный код оборачивается в тройные обратные кавычки. Если после
открывающего маркера указать имя языка, оно не попадет в текст сообщения.

.. code-block:: python

   await message.answer(
       """```python
   print("hello")
   ```"""
   )

Этот фрагмент будет отправлен как текст ``print("hello")`` с элементом
``CODE``.

Поддерживаемый HTML-синтаксис
-----------------------------

Для HTML-разбора передайте ``parse_mode=ParseMode.HTML``.

.. list-table::
   :header-rows: 1
   :widths: 30 30 40

   * - Синтаксис
     - Тип элемента
     - Описание
   * - ``<b>`` или ``<strong>``
     - ``STRONG``
     - Жирный текст.
   * - ``<i>`` или ``<em>``
     - ``EMPHASIZED``
     - Выделенный текст.
   * - ``<ins>`` или ``<u>``
     - ``UNDERLINE``
     - Подчеркнутый текст.
   * - ``<del>`` или ``<s>``
     - ``STRIKETHROUGH``
     - Зачеркнутый текст.
   * - ``<pre>`` или ``<code>``
     - ``MONOSPACED``
     - Моноширинный текст.
   * - ``<a href="https://example.com">текст</a>``
     - ``LINK``
     - Ссылка с отображаемым текстом.
   * - ``<h1>`` ... ``<h6>``
     - ``HEADING``
     - Заголовок.
   * - ``<blockquote>``
     - ``QUOTE``
     - Цитата.

Ограничения
-----------

Это не полный Markdown- или HTML-парсер. Форматтер поддерживает только
перечисленные маркеры из ``pymax.formatting.parse_text()``. HTML-синтаксис
взят как совместимая основа из Bot API MAX, но PyMax для userbot-транспорта
сам преобразует его в ``Element``.

Обычные inline-маркеры не продолжаются через перенос строки. Если закрывающий
маркер не найден в той же строке или внутри маркеров пустой текст, маркер
остается обычным текстом.

Экранирование служебных символов обратным слешем пока не обрабатывается.
